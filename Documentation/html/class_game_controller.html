Detailed Description

The brains behind the entire Game.

For pragmatic reasons, this class doesn't follow MVC completely. A single "view" cannot really exist for the whole game. There is the GameController, and its model _gameState (the GameXML class). The other controllers in the game such as the InventoryController, control their own "views". All these views are collectively controlled by GameController, instead of having a single view.
